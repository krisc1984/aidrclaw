智能双录
产品需求文档（PRD）

面向 Agent 的下一代智能双录系统

版本：V1.0
目标行业：金融理财、基金、财富产品销售
部署模式：纯私有化部署
核心理念：可插拔、模块化（基于 zeroclaw 思想）、Agent 驱动

---

1. 背景与目标

1.1 业务背景

在财富管理、基金销售等金融场景中，监管要求对销售过程进行“双录”（录音录像），以确保合规留痕、防范销售误导。传统双录系统流程僵化、扩展性差，难以适应复杂产品线和智能化质检需求。随着 AI 技术成熟，引入 Agent 能力可实现自动化引导、实时辅助、智能质检，大幅提升合规效率与用户体验。

1.2 产品目标

打造一个可插拔、模块化、面向 Agent 的智能双录系统：

· 合规存档：完整记录销售过程音视频，满足监管留存要求。
· 实时质检（录制后预质检）：录制完成后快速输出合规检测结果，支持人工复核。
· Agent 自动化交互：通过 AI 虚拟坐席与内部智能模块，动态引导客户完成双录流程，基于规则或 AI 判断跳转流程节点。
· 高度可扩展：所有核心能力（采集、AI、存储、流程引擎）均以插件形式存在，可独立替换或升级。
· 私有化：系统完全部署在企业内部，数据不出域。

3. 总体架构与模块化设计

系统采用微内核 + 插件化架构，核心引擎只提供基础调度与生命周期管理，所有业务功能均由插件实现。插件通过统一接口注册，支持热插拔。

3.1 核心模块划分

```
┌─────────────────────────────────────────────────────┐
│                    双录编排引擎                       │
│   (流程调度、状态机、插件生命周期)                     │
└───────────────┬─────────────────────────────────────┘
                │ 插件总线（统一接口）
┌───────────────┼─────────────────────────────────────┐
│ 采集插件域     │  AI插件域      │  存储插件域      │
│ - 摄像头       │ - 语音识别      │ - 本地文件系统   │
│ - 麦克风       │ - 人脸比对      │ - 对象存储       │
│ - 屏幕录制     │ - 活体检测      │ - 加密归档       │
│ - 移动端采集   │ - 合规质检      │ - 数据库         │
│ - Web采集      │ - Agent对话     │                  │
└───────────────┴─────────────────┴──────────────────┘
                │
┌───────────────┴─────────────────────────────────────┐
│                   Agent 智能体域                     │
│  虚拟坐席Agent、质检Agent、流程决策Agent             │
└─────────────────────────────────────────────────────┘
```

3.2 可插拔设计原则

· 插件标准：定义统一的插件接口（Plugin），包含 init(), execute(), destroy() 等生命周期方法。
· 依赖倒置：核心模块不直接依赖具体插件，通过 SPI（服务提供者接口）发现和加载插件。
· 配置化：所有插件参数通过配置文件或管理后台动态注入。
· 版本隔离：插件可独立打包、部署、升级，互不影响。

4. 功能需求

4.1 双录流程管理（核心引擎）

· 流程模板：支持可视化定义双录流程节点（如：身份核验→风险揭示→产品介绍→确认签字）。每个节点可绑定 Agent 动作或质检规则。
· 状态机：管理双录会话状态（待开始、录制中、预质检中、完成、异常等）。
· 动态跳转：Agent 可根据规则或 AI 判断，跳转至特定节点（例如人脸比对失败则重试，客户回答超时则重复讲解）。

4.2 Agent 智能体域

4.2.1 虚拟坐席 Agent

· 多模态交互：支持文本、语音、视频形象（可选），通过插件化 TTS/ASR 实现。
· 对话流程：基于意图识别 + 规则引擎，引导客户完成双录话术。
· 动态适配：根据客户回答内容，自动调整后续问题（如客户已了解风险，则跳过详细说明）。
· 异常处理：当客户连续答错或情绪异常时，自动转人工或终止流程。

4.2.2 质检 Agent

· 预质检：录制完成后，自动调用质检插件对音视频进行分析，输出合规评分、违规片段、风险提示。
· 质检维度：包含话术完整性、关键话术播报、客户应答合规、人脸在框、无遮挡、无离席等。
· 报告生成：生成可视化质检报告，供合规管理员复审。

4.2.3 流程决策 Agent

· 规则引擎：基于业务规则（如产品风险等级、客户年龄、投资经验）动态调整录制流程。
· AI 辅助决策：可选接入模型判断客户情绪、理解意图，辅助跳转或终止。

4.3 采集插件域

· 移动端采集：支持 iOS/Android App 调用系统摄像头、麦克风，提供音视频流。
· Web 采集：基于 WebRTC 实现浏览器端采集，无需安装插件。
· 屏幕录制：可选插件，用于记录客户操作界面（如基金购买页面）。
· 采集控制：支持动态切换摄像头、麦克风、录制质量参数。

4.4 AI 插件域

· 语音识别（ASR）：将客户语音转文本，用于话术比对和意图理解。
· 语音合成（TTS）：为虚拟坐席生成语音播报，支持多音色、多语种。
· 人脸比对：将客户人脸与身份证照片比对，核验身份。
· 活体检测：动作检测（眨眼、张嘴）或静默活体。
· 合规检测：检测违规词汇、违规动作（如吸烟、接电话）、画面合规（光线、遮挡）。
· 插件扩展：以上能力均可替换为第三方服务（如科大讯飞、阿里云、商汤等）或自研模型。

4.5 存储插件域

· 音视频存储：支持本地磁盘、NAS、分布式对象存储（如 MinIO、Ceph）。
· 加密归档：可选插件，对录制的音视频进行加密存储，满足合规要求。
· 元数据存储：双录记录信息（客户信息、时间、流程结果、质检结果）存储到关系型数据库（如 MySQL、PostgreSQL）或时序数据库。
· 冷热分离：支持将历史记录自动迁移至低成本存储。

4.6 管理后台

· 插件市场：展示已安装插件，支持启用/停用、配置参数、版本升级。
· 流程设计器：拖拽式编排双录流程，绑定 Agent 节点和质检规则。
· 录制记录查询：按客户、时间、产品、合规状态等维度检索，支持音视频在线回放。
· 质检管理：展示预质检结果列表，支持人工复核、修改结论、添加备注。
· 审计日志：记录所有操作日志，满足安全审计要求。

4.7 客户端（App & Web）

· 发起双录：理财经理生成双录邀请（二维码/链接），客户扫码或点击链接进入。
· 引导界面：虚拟坐席以视频或语音形式引导客户完成流程，界面简洁，支持音量调节、重试等。
· 异常提示：当检测到遮挡、离席、噪音过大时，Agent 主动提示客户调整。
· 完成与查看：录制完成后，客户可确认提交，并查看合规状态（通过/待审核）。

5. 非功能需求

5.1 合规与安全

· 数据隐私：所有音视频及客户信息存储在私有化环境，无明文外传。
· 加密传输：客户端与服务器通信采用 TLS 1.3 加密。
· 防篡改：对录制的文件计算哈希值，防止篡改。
· 权限控制：基于角色的访问控制（RBAC），确保只有授权人员可查看敏感数据。

5.2 性能与扩展性

· 并发支持：单节点支持 100+ 并发录制会话，支持水平扩展。
· 预质检延迟：录制完成后，质检结果在 5 分钟内输出（取决于视频长度和 AI 模型处理速度）。
· 插件热插拔：插件更新无需重启核心服务，通过动态类加载实现。

5.3 可用性

· 高可用：核心服务支持多副本部署，无单点故障。
· 容错：客户端网络中断时，支持断点续传或临时本地缓存，恢复后自动续传。

5.4 可维护性

· 日志与监控：提供统一日志输出，支持对接 ELK、Prometheus 等监控系统。
· 插件开发文档：提供详细的插件接口说明和示例代码，便于二次开发。

6. 可插拔插件接口规范（摘要）

6.1 插件生命周期接口

```java
public interface Plugin {
    // 初始化插件，注入配置
    void init(Map<String, Object> config);
    // 执行核心功能（异步）
    CompletableFuture<PluginResult> execute(PluginContext context);
    // 销毁资源
    void destroy();
    // 获取插件元信息
    PluginMetadata getMetadata();
}
```

6.2 采集插件接口示例

```java
public interface CapturePlugin extends Plugin {
    // 开始采集，返回流标识
    String startCapture(CaptureConfig config);
    // 停止采集
    void stopCapture(String streamId);
    // 获取音视频帧（用于实时处理）
    void registerFrameListener(FrameListener listener);
}
```

6.3 AI 插件接口示例

```java
public interface AIPlugin extends Plugin {
    // 处理音视频流或文件
    CompletableFuture<AIResult> analyze(AIRequest request);
    // 支持批处理或流式
}
```

6.4 存储插件接口示例

```java
public interface StoragePlugin extends Plugin {
    // 保存文件
    String save(InputStream data, StorageMetadata metadata);
    // 读取文件
    InputStream load(String path);
    // 删除
    void delete(String path);
}
```

7. 技术选型建议（私有化部署）

组件 推荐技术 说明
核心框架 Spring Boot + Java/Kotlin 成熟稳定，插件化可借助 SPI 或 OSGi（简化版使用自定义类加载器）
流程引擎 Camunda 或自研轻量级状态机 支持 BPMN 2.0，便于可视化编排
数据库 PostgreSQL 存储元数据、流程定义、质检结果
音视频处理 FFmpeg、GStreamer 用于转码、截图、水印等
对象存储 MinIO 兼容 S3，私有化部署简单
消息队列 Apache Kafka 或 RabbitMQ 解耦录制任务与质检任务
前端（Web） React/Vue + WebRTC 实现浏览器端录制
移动端 Flutter 或原生开发 跨平台与性能兼顾

8. 版本规划

V1.0（MVP）

· 完成核心双录流程，支持 App/Web 录制
· 提供基础采集插件（摄像头、麦克风）、本地存储插件
· Agent 实现简单规则驱动的虚拟坐席（固定话术、简单跳转）
· 预质检功能：人脸比对、话术完整性检测（基于关键词）
· 管理后台：录制记录查询、人工复核

V1.5

· 插件机制完善，支持热插拔
· 增加屏幕录制插件、活体检测插件
· Agent 引入意图识别，支持更灵活的对话
· 质检增加违规动作、情绪检测

V2.0

· 完全开放插件市场，支持第三方开发者上传插件
· Agent 引入大模型能力，支持复杂意图理解与动态决策
· 支持多租户（集团内不同子公司隔离）
· 对接更多第三方 AI 服务插件

· 双录：录音录像，金融销售过程合规留痕。

· 可插拔：系统功能模块可独立安装、卸载、替换，不影响其他模块。
