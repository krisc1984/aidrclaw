## 阶段详情

### 阶段 1: 基础架构与插件系统

**目标**: 建立微内核 + 插件化基础架构，实现插件生命周期管理和基础存储能力

**需求映射**:
- FLOW-02: 管理双录会话状态（待开始、录制中、预质检中、完成、异常等）
- STORAGE-01: 支持本地磁盘、NAS、分布式对象存储（如 MinIO、Ceph）多种存储方式
- STORAGE-03: 元数据存储到关系型数据库（MySQL、PostgreSQL）或时序数据库

**成功标准**:
1. 插件接口定义完成（Plugin.init(), execute(), destroy()）
2. 插件加载器可发现和加载插件
3. 会话状态机可正确流转状态
4. 元数据可持久化到数据库
5. 基础存储插件可保存文件到本地磁盘

**UI hint**: no

### 阶段 3: 存储插件域

**目标**: 实现音视频文件存储和加密归档能力

**需求映射**:
- STORAGE-01: 支持本地磁盘、NAS、分布式对象存储（如 MinIO、Ceph）多种存储方式
- STORAGE-02: 加密归档插件对录制的音视频进行加密存储，满足合规要求
- STORAGE-03: 元数据存储到关系型数据库（MySQL、PostgreSQL）或时序数据库

**成功标准**:
1. 音视频文件可保存到本地磁盘
2. 支持 MinIO 对象存储
3. 加密插件可对文件进行 AES 加密
4. 加密文件可正确解密读取
5. 元数据（文件路径、加密状态、时间戳）可查询

**UI hint**: no

---

### 阶段 4: AI 插件域基础

**目标**: 实现基础 AI 能力插件（ASR、TTS、人脸比对、活体检测）

**需求映射**:
- AI-01: 语音识别（ASR）插件将客户语音转文本，用于话术比对和意图理解
- AI-02: 语音合成（TTS）插件为虚拟坐席生成语音播报，支持多音色、多语种
- AI-03: 人脸比对插件将客户人脸与身份证照片比对，核验身份
- AI-04: 活体检测插件支持动作检测（眨眼、张嘴）或静默活体

**成功标准**:
1. ASR 插件可将语音转为文本（准确率≥95%）
2. TTS 插件可生成自然语音（支持中文）
3. 人脸比对插件可核验身份（准确率≥99%）
4. 活体检测插件可检测眨眼、张嘴动作
5. 所有 AI 插件可独立替换（如替换为讯飞、阿里云服务）

**UI hint**: no

---

### 阶段 5: 流程引擎

**目标**: 实现双录流程编排引擎，支持可视化定义流程和动态跳转

**需求映射**:
- FLOW-01: 支持可视化定义双录流程节点（身份核验→风险揭示→产品介绍→确认签字）
- FLOW-02: 管理双录会话状态（待开始、录制中、预质检中、完成、异常等）
- FLOW-03: 支持基于规则或 AI 判断的动态流程跳转（人脸比对失败则重试，客户回答超时则重复讲解）

**成功标准**:
1. 可通过配置文件定义流程节点和顺序
2. 流程引擎可执行定义的流程
3. 状态机可正确管理会话状态流转
4. 支持基于规则的条件跳转（如人脸比对失败→重试）
5. 支持超时处理和重复讲解

**UI hint**: yes (流程设计器界面)

---

### 阶段 6: Agent 智能体域

**目标**: 实现虚拟坐席 Agent 和流程决策 Agent，完成智能化引导

**需求映射**:
- AGENT-01: 虚拟坐席 Agent 支持文本、语音多模态交互，通过插件化 TTS/ASR 实现
- AGENT-02: 虚拟坐席 Agent 基于意图识别 + 规则引擎引导客户完成双录话术
- AGENT-04: 流程决策 Agent 根据业务规则（产品风险等级、客户年龄、投资经验）动态调整录制流程

**成功标准**:
1. 虚拟坐席可通过文本和语音与客户交互
2. 可引导客户完成标准双录话术流程
3. 可根据客户回答调整后续问题
4. 流程决策 Agent 可根据产品风险等级调整流程
5. 异常情况下可转人工或终止流程

**UI hint**: yes (虚拟坐席界面)

---

### 阶段 7: 质检 Agent 与合规检测

**目标**: 实现预质检 Agent 和违规检测能力

**需求映射**:
- AGENT-03: 质检 Agent 录制完成后自动调用质检插件分析音视频，输出合规评分、违规片段、风险提示
- AI-05: 合规检测插件检测违规词汇、违规动作（如吸烟、接电话）、画面合规（光线、遮挡）

**成功标准**:
1. 录制完成后自动触发质检流程
2. 质检 Agent 可检测话术完整性
3. 可检测违规词汇（如承诺保本保收益）
4. 可检测违规动作（吸烟、接电话）
5. 可检测画面合规（光线不足、遮挡、离席）
6. 生成可视化质检报告（合规评分、违规片段标记）

**UI hint**: no

---

### 阶段 8: 管理后台与客户端

**目标**: 完成管理后台和客户体验界面

**需求映射**:
- ADMIN-01: 插件市场展示已安装插件，支持启用/停用、配置参数、版本升级
- ADMIN-02: 流程设计器支持拖拽式编排双录流程，绑定 Agent 节点和质检规则
- ADMIN-03: 录制记录查询按客户、时间、产品、合规状态等维度检索，支持音视频在线回放
- ADMIN-04: 质检管理展示预质检结果列表，支持人工复核、修改结论、添加备注
- CLIENT-01: 支持理财经理生成双录邀请（二维码/链接），客户扫码或点击链接进入
- CLIENT-02: 客户引导界面支持虚拟坐席以视频或语音形式引导客户完成流程
- CLIENT-03: 异常提示（遮挡、离席、噪音过大）时 Agent 主动提示客户调整

**成功标准**:
1. 管理员可查看和管理已安装插件
2. 可通过拖拽方式设计双录流程
3. 可按多维度查询录制记录
4. 支持在线回放音视频
5. 合规管理员可复核质检结果并修改结论
6. 理财经理可生成双录邀请链接/二维码
7. 客户可通过链接进入并完成双录流程
8. 异常情况（遮挡、离席、噪音）主动提示客户

**UI hint**: yes (管理后台 + 客户端界面)
