# 智能双录系统

## What This Is

**下一代智能双录系统** — 面向金融理财、基金、财富产品销售场景的可插拔、模块化、Agent 驱动的双录平台。

**核心定位**：纯私有化部署的企业级合规双录系统，通过 Agent 自动化和智能质检实现销售过程的合规留痕与高效审核。

---

## Context

### 业务背景

在财富管理、基金销售等金融场景中，监管要求对销售过程进行"双录"（录音录像），以确保合规留痕、防范销售误导。传统双录系统存在以下痛点：

- **流程僵化**：固定流程难以适应复杂产品线和多样化销售场景
- **扩展性差**：功能升级需要整体改造，无法独立替换模块
- **质检低效**：依赖人工审核，效率低下且标准不一
- **智能化不足**：缺乏 AI 辅助引导和实时合规检测能力

### 产品愿景

打造一个**可插拔、模块化、面向 Agent** 的智能双录系统：

- **合规存档**：完整记录销售过程音视频，满足监管留存要求
- **实时质检**：录制完成后快速输出合规检测结果，支持人工复核
- **Agent 自动化**：通过 AI 虚拟坐席动态引导客户完成双录流程
- **高度可扩展**：所有核心能力（采集、AI、存储、流程引擎）均以插件形式存在，可独立替换或升级
- **私有化部署**：系统完全部署在企业内部，数据不出域

## Target Users

| 角色 | 核心职责 | 使用场景 |
|------|----------|----------|
| **客户** | 接受双录 | 通过 App 或 Web 与虚拟坐席交互，完成产品销售过程录制 |
| **理财经理** | 辅助发起、查看质检结果 | 在客户录制时实时查看流程状态，接收预质检报告 |
| **合规管理员** | 审核双录记录 | 对预质检异常的记录进行人工复审，标记合规结论 |
| **系统管理员** | 配置插件、流程、规则 | 部署插件、定义双录流程、设置 Agent 对话逻辑 |
| **插件开发者** | 扩展系统能力 | 基于开放接口开发新的采集、AI、存储等插件 |

## Core Value

**单一核心价值**：通过 Agent 驱动的自动化双录流程，将金融销售合规双录的效率和体验提升到新水平，同时保证 100% 监管合规。

**核心价值验证指标**：
- 双录流程完成率 ≥ 95%
- 预质检自动化覆盖率 ≥ 80%
- 人工复核工作量减少 ≥ 60%
- 客户双录体验满意度 ≥ 4.5/5

## Constraints

### 技术约束

| 约束 | 影响 |
|------|------|
| **纯私有化部署** | 所有组件必须支持本地部署，不得依赖公有云服务 |
| **数据不出域** | 音视频及客户信息必须存储在私有环境，无明文外传 |
| **合规安全** | TLS 1.3 加密传输、文件哈希防篡改、RBAC 权限控制 |
| **插件化架构** | 所有核心能力必须以插件形式实现，支持热插拔 |

### 非功能需求

| 维度 | 目标 |
|------|------|
| **并发支持** | 单节点支持 100+ 并发录制会话，支持水平扩展 |
| **预质检延迟** | 录制完成后 5 分钟内输出质检结果 |
| **高可用** | 核心服务支持多副本部署，无单点故障 |
| **容错** | 客户端网络中断时支持断点续传或本地缓存 |

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| **微内核 + 插件化架构** | 传统双录系统扩展性差，功能升级需整体改造 | 核心引擎只提供基础调度，所有业务功能由插件实现 |
| **Agent 驱动流程** | 固定流程无法适应复杂场景，需要动态跳转能力 | 引入虚拟坐席 Agent、质检 Agent、流程决策 Agent |
| **预质检模式** | 实时质检技术复杂度高，预质检可快速落地 | 录制完成后自动质检，输出报告供人工复核 |
| **Java/Spring Boot 技术栈** | 金融企业现有技术栈以 Java 为主，便于私有化部署 | 核心框架采用 Spring Boot，插件化借助 SPI 机制 |

### Active

#### 双录流程管理

- [ ] **FLOW-01**：支持可视化定义双录流程节点（身份核验→风险揭示→产品介绍→确认签字）
- [ ] **FLOW-02**：管理双录会话状态（待开始、录制中、预质检中、完成、异常等）
- [ ] **FLOW-03**：支持基于规则或 AI 判断的动态流程跳转

#### Agent 智能体域

- [ ] **AGENT-01**：虚拟坐席 Agent 支持文本、语音多模态交互
- [ ] **AGENT-02**：虚拟坐席 Agent 基于意图识别 + 规则引擎引导客户完成话术
- [ ] **AGENT-03**：质检 Agent 自动分析音视频输出合规评分和违规片段
- [ ] **AGENT-04**：流程决策 Agent 根据业务规则动态调整录制流程

#### 采集插件域

- [ ] **CAPTURE-01**：移动端采集插件（iOS/Android）调用系统摄像头、麦克风
- [ ] **CAPTURE-02**：Web 采集插件基于 WebRTC 实现浏览器端采集
- [ ] **CAPTURE-03**：支持动态切换摄像头、麦克风、录制质量参数

#### AI 插件域

- [ ] **AI-01**：语音识别（ASR）插件将语音转文本
- [ ] **AI-02**：语音合成（TTS）插件为虚拟坐席生成语音
- [ ] **AI-03**：人脸比对插件核验客户身份
- [ ] **AI-04**：活体检测插件（动作检测或静默活体）
- [ ] **AI-05**：合规检测插件检测违规词汇、动作、画面合规

#### 存储插件域

- [ ] **STORAGE-01**：支持本地磁盘、NAS、对象存储（MinIO）多种存储方式
- [ ] **STORAGE-02**：加密归档插件对音视频加密存储
- [ ] **STORAGE-03**：元数据存储到关系型数据库（MySQL/PostgreSQL）

#### 管理后台

- [ ] **ADMIN-01**：插件市场展示已安装插件，支持启用/停用、配置参数
- [ ] **ADMIN-02**：流程设计器支持拖拽式编排双录流程
- [ ] **ADMIN-03**：录制记录查询支持多维度检索和在线回放
- [ ] **ADMIN-04**：质检管理展示预质检结果并支持人工复核

#### 客户端

- [ ] **CLIENT-01**：支持理财经理生成双录邀请（二维码/链接）
- [ ] **CLIENT-02**：客户引导界面支持虚拟坐席视频/语音引导
- [ ] **CLIENT-03**：异常提示（遮挡、离席、噪音）主动提示客户调整
