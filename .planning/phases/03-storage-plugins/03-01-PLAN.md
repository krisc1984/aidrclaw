  <name>Task 1: MinIO 插件实现与单元测试</name>
  <files>aidrclaw-plugins-storage/MinioStoragePlugin.java, aidrclaw-plugins-storage/MinioStoragePluginTest.java</files>
  <behavior>
    - Test 1: 插件可初始化并创建 MinioClient 实例
    - Test 2: putObject 操作可上传文件到 MinIO
    - Test 3: getObject 操作可下载文件
    - Test 4: deleteObject 操作可删除文件
    - Test 5: presignedUrl 操作可生成预签名 URL
    - Edge case: MinIO 服务不可用时返回错误而非异常
  </behavior>
  <action>
    实现 MinioStoragePlugin 类，遵循以下规范：

    1. **插件元数据**:
       - pluginId: "minio-storage"
       - version: "1.0.0"
       - name: "MinIO 对象存储插件"
       - description: "支持 S3 兼容协议的对象存储"

    2. **配置参数** (通过 PluginContext.getConfig* 获取):
       - endpoint: MinIO 服务端点 (如 http://localhost:9000)
       - accessKey: 访问密钥
       - secretKey: 密钥
       - bucket: 默认 bucket 名称
       - region: 区域 (默认 us-east-1)

    3. **支持的 action**:
       - "save": 保存文件到 MinIO
         - 输入：{ inputStream, objectKey, contentType, metadata }
         - 输出：{ objectKey, bucket, etag, size }
       - "load": 从 MinIO 获取文件
         - 输入：{ objectKey }
         - 输出：{ inputStream, contentType, metadata }
       - "delete": 删除文件
         - 输入：{ objectKey }
         - 输出：{ deleted: true }
       - "presignedUrl": 生成预签名 URL
         - 输入：{ objectKey, expirySeconds }
         - 输出：{ url, expiry }

    4. **实现要点**:
       - 使用 MinioClient.putObject() 上传
       - 使用 MinioClient.getObject() 下载
       - 使用 MinioClient.presignedGetObject() 生成 URL
       - 所有操作包裹在 try-catch 中，返回 PluginResult.error 而非抛异常
       - 日志记录关键操作（使用 SLF4J）

    5. **依赖添加** (pom.xml):
       ```xml
       <dependency>
           <groupId>io.minio</groupId>
           <artifactId>minio</artifactId>
           <version>8.5.7</version>
       </dependency>
       ```

    参考 LocalStoragePlugin 的实现模式，保持接口一致性。
  </action>
  <verify>
    <automated>mvn test -pl aidrclaw-plugins-storage -Dtest=MinioStoragePluginTest -q</automated>
  </verify>
  <done>MinioStoragePlugin 单元测试全部通过，可独立编译加载</done>

       ```java
               .build();
       }
       ```
