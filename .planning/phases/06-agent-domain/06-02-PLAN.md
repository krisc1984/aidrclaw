           id VARCHAR(64) PRIMARY KEY,
           name VARCHAR(255) NOT NULL COMMENT '规则名称',
           category VARCHAR(64) NOT NULL COMMENT '规则分类',
           condition_expression TEXT NOT NULL COMMENT 'SpEL 条件表达式',
           action_json TEXT NOT NULL COMMENT 'JSON 动作定义',
           priority INT DEFAULT 0 COMMENT '优先级 (数字越大优先级越高)',

  <action>
    实现流程决策 Agent 的核心逻辑：

    1. **ProcessDecisionAgent 类**:
       - 注入 BusinessRuleRepository、DecisionEngine、AgentRulesEngine
       - 实现 evaluateContext(DecisionContext context): DecisionResult 方法
       - 流程:
         a. 从 BusinessRuleRepository 查询适用规则
         b. 调用 DecisionEngine.evaluate 评估规则
         c. 收集所有触发规则的动作
         d. 合并动作生成 DecisionResult
         e. 记录决策原因和日志

    2. **DecisionEngine 类**:
       - evaluate(DecisionContext context, List<BusinessRule> rules): List<BusinessRule>
       - 使用 SpEL 解析并评估规则条件
       - 按优先级排序规则
       - 返回所有条件为 true 的规则列表
       - 提供 SpEL 上下文构建方法

    3. **RuleExecutor 类**:
       - execute(BusinessRule rule, DecisionContext context): FlowAdjustment
       - 解析规则的 actionJson
       - 根据动作类型执行不同逻辑:
         - INSERT_NODE: 创建插入节点动作
         - SKIP_NODE: 创建跳过节点动作
         - MODIFY_NODE: 创建修改节点动作
         - CHANGE_FLOW: 创建切换流程动作
       - 返回 FlowAdjustment 对象

    4. **SpEL 条件表达式示例**:
       - "context.product.riskLevel >= 'R4'" - 产品风险等级≥R4
       - "context.customer.age >= 60" - 客户年龄≥60 岁
       - "context.customer.investmentExperience == true" - 有投资经验
       - "context.session.failureCount >= 3" - 失败次数≥3

    5. **JSON 动作示例**:
       ```json
       {
         "type": "INSERT_NODE",
         "nodeId": "EXTRA_RISK_WARNING",
         "position": "AFTER_RISK_DISCLOSURE",
         "parameters": {
           "speed": 0.8,
           "tone": "serious"
         }
       }
       ```

    6. **决策日志**:
       - 记录触发的规则名称
       - 记录决策原因
       - 记录调整后的流程参数
       - 使用 SLF4J 记录到审计日志
  </action>
  <behavior>
    - Test 1: 产品风险等级 R4 时，触发"增加风险揭示"规则
    - Test 2: 客户年龄≥60 岁时，触发"语速降低"规则
    - Test 3: 失败次数≥3 时，触发"转人工"规则
    - Test 4: 多条规则同时触发时，按优先级执行
    - Test 5: 无规则触发时，返回 noAdjustment 结果
  </behavior>
