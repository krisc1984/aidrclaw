// 对话节点类型
public enum NodeType {
    MESSAGE,    // 显示消息
    QUESTION,   // 提问并分支
    ACTION,     // 执行动作
    MERGE,      // 汇聚点
    TERMINAL    // 对话结束
}

// 对话节点
public class DialogNode {
    private String id;
    private NodeType type;
    private String content;
    private QuestionConfig question;
    private ActionConfig action;
    private Map<String, String> children; // 意图→下一节点 ID
}

// 对话树
public class DialogTree {
    private String id;
    private String name;
    private String rootNodeId;
    private Map<String, DialogNode> nodes;
    private DialogTreeMetadata metadata;
}

  <action>
    实现虚拟坐席核心服务，整合对话树和规则引擎：

    1. **VirtualAgentService 类**:
       - 注入 DialogTreeManager、AgentRulesEngine、TtsPlugin、AsrPlugin
       - 实现 processTurn(String sessionId, UserInput input): AgentResponse 方法
       - 流程:
         a. 从 DialogTreeManager 获取当前对话状态
         b. 根据 UserInput 更新会话上下文
         c. 调用 AgentRulesEngine.fireRules 执行规则
         d. 根据意图和规则结果确定下一节点
         e. 从 DialogTreeManager 获取下一节点内容
         f. 调用 TtsPlugin.synthesize 生成语音
         g. 返回 AgentResponse (文本 + 语音)

    2. **AgentResponse 类**:
       - text: String - 文本回复
       - audioUrl: String - 语音 URL 或 Base64
       - nextNodeId: String - 下一节点 ID
       - metadata: Map - 元数据 (意图、置信度等)

    3. **UserInput 类**:
       - type: InputType (TEXT, AUDIO)
       - text: String - 文本输入
       - audioData: byte[] - 音频数据
       - timestamp: Instant - 时间戳

    4. **意图识别逻辑**:
       - 文本输入：关键词匹配 (包含"是"/"对"→CONFIRM, "不"/"没"→DENY)
       - 音频输入：先调用 AsrPlugin.recognize 转为文本，再关键词匹配
       - 超时：超过设定时间无输入→TIMEOUT 意图

    5. **异常处理**:
       - 节点不存在：抛出 DialogNodeNotFoundException
       - TTS 失败：返回文本响应，记录错误日志
       - ASR 失败：提示客户重新输入
  </action>
  <behavior>
    - Test 1: 文本输入"是的"时，意图识别为 CONFIRM，流转到下一节点
    - Test 2: 文本输入"不是"时，意图识别为 DENY，触发重试逻辑
    - Test 3: 音频输入时，先调用 ASR 转文本，再识别意图
    - Test 4: TTS 失败时，AgentResponse 包含 text 但 audioUrl 为空
    - Test 5: 节点不存在时，抛出 DialogNodeNotFoundException
  </behavior>
