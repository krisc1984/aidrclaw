# Phase 6: Agent 智能体域 - 上下文与目标

## 阶段目标

实现虚拟坐席 Agent 和流程决策 Agent，完成智能化引导。

## 需求映射

**Requirements**: [AGENT-01, AGENT-02, AGENT-04]

### AGENT-01: 虚拟坐席多模态交互
虚拟坐席 Agent 支持文本、语音多模态交互，通过插件化 TTS/ASR 实现。

**验收标准**:
- 虚拟坐席可通过文本与客户交互
- 虚拟坐席可通过语音（TTS）与客户交互
- 客户可通过文本回复
- 客户可通过语音（ASR）回复
- TTS/ASR 通过插件化集成（可替换）

### AGENT-02: 意图识别 + 规则引擎引导
虚拟坐席 Agent 基于意图识别 + 规则引擎引导客户完成双录话术。

**验收标准**:
- 可识别客户意图（确认、否认、疑问、跳过等）
- 基于规则引擎引导标准双录话术流程
- 可根据客户回答调整后续问题
- 支持超时处理和重复讲解

### AGENT-04: 流程决策 Agent
流程决策 Agent 根据业务规则（产品风险等级、客户年龄、投资经验）动态调整录制流程。

**验收标准**:
- 可根据产品风险等级（R1-R5）调整流程
- 可根据客户年龄调整语速和流程复杂度
- 可根据投资经验调整风险揭示详细程度
- 异常情况可转人工或终止流程

## 技术栈建议

### 对话管理
- **规则引擎**: Drools / EasyRules / LiteFlow
- **意图识别**: 基于规则的模式匹配（V1.0）→ 后续可升级为 ML 模型
- **对话状态跟踪**: 基于有限状态机或对话树

### Agent 架构
- **虚拟坐席 Agent**: 对话管理器 + TTS/ASR 插件集成
- **流程决策 Agent**: 规则引擎 + 业务规则配置

### 界面需求 (UI hint: yes)
- 虚拟坐席对话界面（视频/音频 + 文本）
- 客户输入区域（语音按钮 + 文本输入框）
- 流程进度指示器

## 与 Phase 4/5 的关系

**依赖 Phase 4 (AI 插件)**:
- 使用 ASR 插件进行语音识别
- 使用 TTS 插件生成语音播报
- 使用人脸比对插件进行身份核验

**依赖 Phase 5 (流程引擎)**:
- 流程决策 Agent 调用流程引擎执行流程
- 对话管理器根据流程节点生成话术

## 关键决策点

### D-01: 意图识别方案
**选项**:
- A: 基于规则的关键词匹配（V1.0 推荐）
- B: 基于 ML 模型的意图分类（V1.5）
- C: 基于大模型的语义理解（V2.0）

**推荐**: 选项 A - V1.0 使用规则驱动，降低复杂度，保证可控性

### D-02: 对话管理架构
**选项**:
- A: 有限状态机（FSM）
- B: 对话树（Dialog Tree）
- C: 基于事件的架构

**推荐**: 选项 B - 对话树更适合话术引导场景，便于配置和维护

### D-03: 流程决策规则存储
**选项**:
- A: 硬编码在 Java 代码中
- B: YAML/JSON 配置文件
- C: 数据库存储（支持动态配置）

**推荐**: 选项 C - 数据库存储，支持管理员在后台动态调整规则

## 成功标准

1. ✅ 虚拟坐席可通过文本和语音与客户交互
2. ✅ 可引导客户完成标准双录话术流程
3. ✅ 可根据客户回答调整后续问题
4. ✅ 流程决策 Agent 可根据产品风险等级调整流程
5. ✅ 异常情况下可转人工或终止流程

## 约束条件

- **纯私有化部署**: 所有 Agent 组件必须支持本地部署
- **V1.0 范围**: 意图识别使用规则驱动，不引入大模型
- **插件化**: TTS/ASR 必须通过插件集成，支持替换
- **合规性**: 对话记录必须完整保存，不可篡改
