  <files>src/main/java/com/aidrclaw/core/process/TimeoutHandler.java</files>
  <behavior>
    - Test 1: 可为节点调度超时定时器
    - Test 2: 超时后触发回调函数
    - Test 3: 可在节点完成前取消定时器
    - Test 4: 支持自定义超时时间（秒）
  </behavior>
  <action>
    实现超时处理机制：
    
    1. **TimeoutHandler.java**:
       - 使用 ScheduledExecutorService 调度定时器
       - 方法：
         - scheduleTimeout(String sessionId, String nodeId, int timeoutSeconds, Runnable onTimeout): ScheduledFuture<?>
           - 创建定时器
           - 超时后执行 onTimeout 回调
           - 返回 ScheduledFuture 用于取消
         - cancelTimeout(ScheduledFuture<?> future): void
           - 取消定时器
         - shutdown(): void
           - 关闭调度器（应用关闭时调用）
       
       - 超时回调逻辑：
         ```java
         onTimeout.run();
         // 在 ProcessEngine 中实现：
         // 1. 查找节点的 onTimeout 跳转配置
         // 2. 调用 transitionTo(onTimeoutNextNodeId)
         ```
    
    2. **与 ProcessEngine 集成**:
       - 在 executeNode() 开始时调用 scheduleTimeout()
       - 节点完成后调用 cancelTimeout()
       - 超时回调中调用 transitionTo() 跳转到超时处理节点
    
    3. **配置支持**:
       - ProcessNode 添加 timeout 字段（秒）
       - 添加 onTimeout 字段（超时跳转节点 ID）
  </action>

<task type="auto" tdd="true">
  <name>Task 2: 实现重试管理器</name>
  <files>src/main/java/com/aidrclaw/core/process/RetryManager.java</files>
  <behavior>
    - Test 1: 可判断是否应该重试（retryCount < maxRetries）
    - Test 2: 可调度延迟重试（delay 毫秒后）
    - Test 3: 可记录每个节点的重试次数
    - Test 4: 达到最大重试次数后返回 false
  </behavior>
  <action>
    实现重试机制：
    
    1. **RetryManager.java**:
       - 使用 ScheduledExecutorService 调度延迟重试
       - 依赖注入：ProcessEngine
       - 方法：
         - shouldRetry(ProcessInstance instance, ProcessNode node): boolean
           - 检查 retryCount < maxRetries
           - 返回是否可重试
         - scheduleRetry(ProcessInstance instance, ProcessNode node, int delayMs): void
           - 延迟 delayMs 后重新执行节点
           - 增加 retryCount
         - getRetryCount(ProcessInstance instance, String nodeId): int
         - resetRetryCount(ProcessInstance instance, String nodeId): void
       
       - 重试逻辑：
         ```java
         if (shouldRetry(instance, node)) {
             scheduleRetry(instance, node, node.getRetryDelay());
         } else {
             // 跳转到失败处理节点
             transitionTo(instance, findFailedTransition(node));
         }
         ```
    
    2. **与 ProcessEngine 集成**:
       - 节点执行失败时调用 RetryManager.shouldRetry()
       - 可重试时调用 scheduleRetry()
       - 不可重试时跳转到失败处理节点（如 manual-review）
    
    3. **配置支持**:
       - ProcessNode 添加 maxRetries 字段
       - ProcessNode 添加 retryDelay 字段（毫秒）
  </action>
  <verify>
    <automated>mvn test -Dtest=RetryManagerTest -pl aidrclaw-core</automated>
  </verify>
  <done>重试管理器可根据配置自动重试失败节点</done>
</task>
