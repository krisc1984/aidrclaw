  <name>Task 4: 实现流程引擎核心</name>
  <files>src/main/java/com/aidrclaw/core/process/ProcessEngine.java, src/main/java/com/aidrclaw/core/process/TransitionManager.java</files>
  <action>
    实现流程引擎核心逻辑：
    
    1. **ProcessEngine.java**:
       - 依赖注入：ProcessDefinitionLoader, NodeExecutor, RuleEngine, PluginManager
       - 方法：
         - startProcess(String processId, String sessionId): ProcessInstance
           - 加载流程定义
           - 创建流程实例
           - 执行第一个节点
         - executeNode(ProcessInstance instance): void
           - 获取当前节点
           - 调用 NodeExecutor.executeNode()
           - 根据结果和规则决定下一节点
           - 更新会话状态
         - transitionTo(ProcessInstance instance, String nextNodeId): void
           - 更新当前节点 ID
           - 更新会话状态机
           - 记录状态历史
       
       - 状态流转逻辑：
         ```java
         PluginResult result = nodeExecutor.executeNode(node, instance);
         instance.getContext().setNodeResult(node.getNodeId(), result);
         
         String nextNodeId = transitionManager.findNextNode(node, result, instance);
         if (nextNodeId != null) {
             transitionTo(instance, nextNodeId);
             executeNode(instance); // 递归执行下一节点
         }
         ```
    
    2. **TransitionManager.java**:
       - 依赖注入：RuleEngine
       - 方法：
         - findNextNode(ProcessNode currentNode, PluginResult result, ProcessInstance instance): String
           - 遍历 currentNode.transitions
           - 使用 RuleEngine 评估每个 transition.condition
           - 返回第一个匹配条件的 next 节点 ID
           - 如果无匹配，返回 null（流程暂停）
         - executeTransitionAction(Transition transition, ScriptContext context): void
           - 执行跳转动作（如 retryCount++）
    
    3. **与 SessionStateMachine 集成**:
       - 每个节点执行前更新会话状态
       - 使用 CONTEXT.md 中定义的状态枚举
  </action>
