  <name>Task 1: 创建流程定义模型</name>
  <files>src/main/java/com/aidrclaw/core/process/ProcessDefinition.java, src/main/java/com/aidrclaw/core/process/ProcessNode.java, src/main/java/com/aidrclaw/core/process/NodeType.java, src/main/java/com/aidrclaw/core/process/Transition.java</files>
  <behavior>
    - Test 1: ProcessDefinition 可从 YAML 解析，包含 processId, name, version, nodes
    - Test 2: ProcessNode 包含 nodeId, name, nodeType, plugin, timeout, maxRetries, transitions
    - Test 3: NodeType 枚举包含 AI_NODE, AGENT_NODE, MANUAL_NODE, CONDITION_NODE, END_NODE
    - Test 4: Transition 包含 condition, next, action 字段
  </behavior>
  <action>
    创建流程定义的核心模型类：
    
    1. **ProcessDefinition.java**: 流程定义根对象
       - 字段：processId (String), name (String), version (String), description (String), nodes (List<ProcessNode>)
       - 使用 Jackson @JsonProperty 注解支持 YAML 解析
       - 添加 getter/setter
    
    2. **ProcessNode.java**: 流程节点定义
       - 字段：nodeId (String), name (String), nodeType (NodeType), plugin (String), timeout (int, 秒), maxRetries (int), retryDelay (int, 毫秒), transitions (List<Transition>)
       - 使用 Jackson @JsonProperty 注解
       - 添加 getter/setter
    
    3. **NodeType.java**: 节点类型枚举
       - 枚举值：AI_NODE, AGENT_NODE, MANUAL_NODE, CONDITION_NODE, END_NODE
       - 每个类型添加注释说明用途
    
    4. **Transition.java**: 节点跳转规则
       - 字段：condition (String, 脚本表达式), next (String, 下一节点 ID), action (String, 可选的执行动作)
       - 使用 Jackson @JsonProperty 注解
       - 添加 getter/setter
    
    参考 CONTEXT.md 中的配置示例结构
  </action>
  <verify>
    <automated>mvn test -Dtest=ProcessDefinitionTest -pl aidrclaw-core</automated>
  </verify>
  <done>流程定义模型类创建完成，支持从 YAML 解析</done>
