- 设备选择：自动选择系统默认摄像头/麦克风

  <name>Task 1: 实现设备枚举与选择</name>
  <files>aidrclaw-web-capture-frontend/src/hooks/useDevices.ts, aidrclaw-web-capture-frontend/src/components/DeviceSelector.tsx</files>
  <behavior>
    - Test: enumerateDevices() 可获取所有音视频设备
    - Test: 可过滤出摄像头设备列表 (kind: 'videoinput')
    - Test: 可过滤出麦克风设备列表 (kind: 'audioinput')
    - Test: 设备选择后可返回对应 deviceId
    - Test: 设备断开时可触发 ondevicechange 事件
  </behavior>
  <action>
    1. 实现 useDevices hook：
       - 调用 navigator.mediaDevices.enumerateDevices() 获取设备列表
       - 过滤出 videoinput（摄像头）和 audioinput（麦克风）
       - 监听 devicechange 事件，设备变化时重新枚举
       - 返回 { videoDevices, audioDevices, selectedVideoId, selectedAudioId, selectVideo, selectAudio }
    2. 实现 DeviceSelector 组件：
       - 两个下拉选择框（摄像头、麦克风）
       - 显示设备友好名称（如 "Built-in iSight Camera"）
       - 选择后更新 selectedVideoId/selectedAudioId
    3. 设备断开检测：
       - 在 useDevices 中监听 navigator.mediaDevices.ondevicechange
       - 设备断开时检查当前选中设备是否还在列表中
       - 不在则显示"设备已断开，请重新选择"提示
  </action>
  <verify>
    <automated>npm test -- --filter=useDevices</automated>
  </verify>
  <done>设备枚举和选择功能完成，可检测断开并提示</done>

<task type="auto">
  <name>Task 3: 集成设备断开处理</name>
  <files>aidrclaw-web-capture-frontend/src/hooks/useWebRTC.ts, aidrclaw-mobile-capture/src/hooks/useCamera.ts</files>
  <action>
    1. Web 端设备断开处理（useWebRTC）：
       - 监听 MediaStream 的 track.onended 事件
       - 检测到 track 结束时暂停录制
       - 显示模态框："摄像头已断开，请重新连接后继续"
       - 提供"重新连接"按钮，重新调用 getUserMedia
    2. 移动端设备断开处理（useCamera）：
       - 监听 App 生命周期（AppState）
       - App 切到后台时暂停 MediaStream（调用 track.stop()）
       - App 返回前台时恢复 MediaStream（重新调用 getUserMedia）
       - 显示提示："录制已暂停，点击继续"
    3. 统一错误提示 UI：
       - 创建 Toast 组件显示临时提示
       - 创建 Modal 组件显示阻断性错误
       - 错误文案友好且包含解决建议
